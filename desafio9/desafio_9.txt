a) Padrões de projetos são formas de estruturar o código de forma padronizada afim de encontrar soluções para problemas comuns em programas orientados a objetos, como o próprio nome diz, o intuito de adotar essas arquiteuras em softwares é trazer alguma solução já existente para problemas que se repetem na produção de sistemas, seja organização do código ou remover limitações que possam existir.

b)I) Padrão de criação visa abstrair o instanciamento de objetos, tem como objetivo deixar o código mais flexivel e eficiente, ou seja, se um projeto necessitar de um objeto ele terá pronto para uso, sem se preocupar como é feita a criação de tal objeto.

II) Padrões estruturais visam em organizar da melhor forma as estruturar e as classes e relações entre classe/objetos dentro do projeto.

III) Padrões comportamentais visam focar em como cada parte do código deve se comportar, qual a responsabilidade dentro do proejto, como irá fazê-lo e se como se comunicar.

c)Arquitetura de software demonstra como organizar e dar sentido a cada componente de um sistema, seguir desde o início uma boa arquitetura em mente, deixa o código sair de forma menos desastroza, planejar e conectar cada componente ao outro dá o sentido e coesão necessária para criar um projeto robusto e escalável futuramente.

d)Modelo usado foi em camadas.
